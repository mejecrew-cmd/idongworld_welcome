## 스토리지 방식 — **하이브리드** (질문하신 “테이블 vs JSON”의 답)
키 수가 적고 단순한 **UI 문구**와, 행이 많고 구조적인 **시계 가사**는 성격이 달라 분리했습니다.

| 데이터 | 방식 | 이유 |
|---|---|---|
| **UI 스트링** (타이틀·버튼·사업자표기 등) | 언어별 **키-값 객체** (`i18n.js`, `data-i18n` 연결) | 키 적음·단순, 레퍼런스(VANNEE) 방식 계승 |
| **K-pop 시계 가사** | **행 데이터 테이블** (`clock.js`) | 행 수 많음·구조적(`slot/num/text/emphasis/source`)·비개발자가 시트로 관리 |

> 프로덕션 전환: `.js` → 순수 `.json` 파일 + `fetch`, 또는 구글시트 CSV/TSV를
> Papa.parse(레퍼런스 `script.js` 패턴)로 불러오면 동일 구조로 동작합니다.

### 시계 가사 테이블 스키마 (`data/clock.js`)
```js
{ slot: "hour" | "minute",  // 시 / 분
  num: 0,                    // hour 0~24, minute 0~59
  text: "한 해가 또 가네",    // 노랫말 (한글 10~15자 권장)
  emphasis: "한 해",          // 숫자말+객체 → 빨강·2배 강조
  source: "곡명 · 그룹 · 앨범" }
```
- 같은 `num`에 행을 여러 개 넣으면 다양하게 랜덤 출력 (같은 분 동안은 고정).
- 다른 언어는 `ko` 옆에 `"en": [...]` 블록 추가.

## 각색 메모 (Air → 아이동월드)
- 배경: sky canvas(#426188)에 위(밝음)→아래(깊음) 하늘 그라데이션 + 흐르는 빛무리 구름.
- 강조색: UI 인터랙션은 Air의 단일 액센트 `#2b7fff`(준비중 버튼/활성 별)만 사용.
  단, **시계의 큰 숫자말은 기획 요청대로 빨강(`--clock-emphasis`)** — 콘텐츠 강조 전용으로 분리.
  (블루로 통일하고 싶으면 `css/style.css`의 `--clock-emphasis` 한 줄만 교체)
- 시계 레이아웃: 숫자말(`emphasis`)을 큰 글씨 포인트로, 가사 전체(`text`)·출처(`source`)는 그 아래 작은 두 줄로.
- 시계 숫자말 색: 보라→마젠타→핑크→빨강 K-pop 그라데이션(`--clock-gradient`) + 12초 느린 시머(`kpopShimmer`).
  단색으로 되돌리려면 `.clock__big`의 그라데이션 대신 `color:var(--clock-emphasis)`. 시머 끄려면 `animation` 한 줄 제거.
- 언어 선택: 2열 그리드(`.starfield`). 1열로 되돌리려면 `grid-template-columns:1fr`.
- 별빛 언어선택: 어두운 남색 배경의 레퍼런스 연출을, 하늘 캔버스 위 흰 별빛으로 각색.
